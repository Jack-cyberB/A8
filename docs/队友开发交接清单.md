# 队友开发交接清单

## 1. 目的

本清单用于把 A8 项目交接给队友继续开发，避免出现以下问题：

- 代码拉下来但分支不对
- MySQL 没数据，后端起不来
- RAGFlow 数据集和 chat 没建，知识问答不可用
- 拿到仓库后不知道先验证什么

当前项目默认开发基线：

- 代码主线：`origin/codex/a8`
- 默认存储：`MySQL`
- 知识问答：`RAGFlow + 本地知识上传包`

## 2. 交接对象

建议交给队友的内容固定为 4 类：

1. 代码分支
2. MySQL 数据
3. RAGFlow 知识库上传包
4. 本地环境变量模板

不建议直接交接的内容：

- 你本机 `.env` 里的真实密钥
- 你本机 RAGFlow 的 `dataset id / chat id`
- 你本机数据库账号密码截图或临时复制内容

## 3. 代码交接

### 3.1 队友拿代码的推荐方式

如果队友是新机器，建议直接 clone 仓库后切到当前开发分支：

```powershell
git clone https://github.com/Jack-cyberB/A8.git
cd A8
git fetch origin
git switch -c local-codex-a8 origin/codex/a8
```

如果队友已经有仓库：

```powershell
git fetch origin
git switch -C local-codex-a8 origin/codex/a8
```

说明：

- `sync/codex-a8` 是当前你本地原目录用的同步分支，只是本地习惯名
- 远端真正需要对齐的是 `origin/codex/a8`

## 4. MySQL 交接

当前项目默认后端是 MySQL，数据库名为 `a8`。

### 4.1 方案 A：直接导出你当前数据库状态

适合要让队友拿到和你完全一致的运行态：

```powershell
mysqldump -uroot -proot --databases a8 > a8_handoff.sql
```

队友导入：

```powershell
mysql -uroot -proot < a8_handoff.sql
```

优点：

- 数据状态与你当前机器最一致
- 适合联调、复现实验结果、保留 AI 日志和备注

### 4.2 方案 B：让队友本地重建

适合只需要基础开发环境：

```powershell
python -m pip install -r requirements.txt
python scripts/init_mysql_storage.py
```

这会创建并导入：

- `buildings`
- `energy_timeseries`
- `weather_timeseries`
- `peer_energy_daily`
- `anomaly_actions`
- `anomaly_notes`
- `ai_calls`
- `system_snapshots`

### 4.3 当前建议

如果队友主要负责“接入其他能耗类型”，优先用 **方案 B** 就够了。  
如果队友需要完全复现你当前本机的运行状态，再用 **方案 A**。

## 5. RAGFlow 交接

### 5.1 需要交接的不是 ID，而是知识包

队友需要使用仓库内现成的知识包目录：

- [sikong-kb-pack](/D:/Project/2026/A8/docs/ragflow/sikong-kb-pack)
- [standard-kb-pack](/D:/Project/2026/A8/docs/ragflow/standard-kb-pack)

其中：

- `sikong-kb-pack` = 场景运维库
- `standard-kb-pack` = 标准规范库

### 5.2 队友自己的 RAGFlow 环境需要自己新建

队友本机需要：

1. 安装并启动自己的 RAGFlow
2. 新建 2 个数据集
3. 分别导入 `sikong-kb-pack` 和 `standard-kb-pack`
4. 自己新建一个 chat
5. 将他本机生成的新 `dataset id / chat id` 写入自己的 `.env`

结论：

- 可交接：知识内容、上传包、目录结构
- 不可直接复用：你本机的 `dataset id / chat id`

### 5.3 如果队友当前只做多能耗开发

如果队友当前阶段只做：

- `chilledwater`
- `water`
- `gas`
- 分析接口
- 前端图表与筛选

那么可以先 **不配置 RAGFlow**，只先把代码和 MySQL 跑通。

## 6. 环境变量交接

仓库已提供模板：

- [.env.example](/D:/Project/2026/A8/.env.example)

队友本地操作：

```powershell
Copy-Item .env.example .env
```

然后按他自己的环境填写：

- `MYSQL_*`
- `OPENAI_*`
- `RAGFLOW_*`

注意：

- 不要把你自己的真实 `.env` 直接提交仓库
- 不要把真实密钥写进文档

## 7. 最低启动步骤

队友拿到仓库后，按以下顺序执行即可：

```powershell
python -m pip install -r requirements.txt
Copy-Item .env.example .env
python scripts/init_mysql_storage.py
python backend/server.py
```

浏览器打开：

- `http://127.0.0.1:8000`

## 8. 最低验证命令

建议交接完成后先跑这几条：

```powershell
python -m unittest backend.tests.test_repository
python scripts/smoke_test_api.py
```

如果队友配置了 RAGFlow 和 DeepSeek，还可以进一步验证：

```powershell
python scripts/run_backend_tests.py
```

## 9. 当前推荐分工边界

为了降低冲突，当前建议按职责改动：

### 队友负责

## 10. 最终建议

当前项目最稳的交接顺序是：

1. 先交代码分支 `origin/codex/a8`
2. 再交 MySQL 初始化方式
3. RAGFlow 只交知识包目录，不交本机 ID
4. 队友先跑通后端和分析页
5. 做多能耗开发时，知识问答链路后补

这样最适合当前 A8 的比赛节奏，也最不容易因为环境问题耽误开发。
